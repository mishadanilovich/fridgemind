# Промпт-добавка для Claude Design — фикс избранного, шаблонов и импорта

Отправь это следующим сообщением в тот же чат Claude Design — это правки уже сделанных экранов (избранное, шаблоны меню, импорт рецептов), не новые фичи. Макет разошёлся со спецификацией продукта в нескольких местах — ниже перечислено, что именно поправить.

---

Поправь в прототипе то, что было сделано для избранного, шаблонов меню и импорта рецептов — сейчас есть расхождения со спецификацией.

## 1. Избранное — сердечко только тем, кто может его переключать

Сейчас сердечко на карточке рецепта и на экране просмотра рецепта показывается всем и нажимается всеми. Переключать избранное может только Организатор/Редактор.

- В виде **"Участник"** избранного нет вообще: ни сердечка на карточке рецепта в списке, ни сердечка на экране просмотра рецепта, ни фильтр-чипа **"Избранное"** рядом с поиском. Место под них не резервируется — заголовок и мета-строка карточки занимают всю ширину, а поле поиска растягивается на всю ширину строки, как если бы чипа не было в макете.
- В виде "Организатор/Редактор" всё остаётся как сейчас.

## 2. Шаблоны меню — нужен экран управления, а не только применение

Сейчас в шите "Меню недели" есть "Сохранить как шаблон" и "Применить шаблон", но шаблон невозможно **переименовать или удалить** — а удаление обязательно, потому что шаблонов не может быть больше четырёх.

- Переименуй второй пункт шита с "Применить шаблон" на **"Шаблоны"** — он открывает полноценный экран управления шаблонами (не просто выбор).
- На экране "Шаблоны" — список не более чем из 4 карточек. На карточке: название, **дата создания** и число запланированных приёмов пищи мелким текстом. У каждой карточки три действия: основная кнопка **"Применить"** и две иконки — **"Переименовать"** и **"Удалить"**.
- "Переименовать" открывает такой же простой диалог с полем ввода, как "Сохранить как шаблон".
- "Удалить" открывает подтверждение с названием шаблона в тексте (в том же стиле, что подтверждение удаления рецепта) — действие необратимое.
- Пустое состояние экрана ("Пока нет шаблонов") оставь как есть.

### Лимит в 4 шаблона

Добавь состояние, когда шаблонов уже 4: по нажатию **"Сохранить как шаблон"** вместо диалога ввода названия показывается сообщение **"Достигнут лимит в 4 шаблона, удалите один, чтобы сохранить новый"** с кнопкой, ведущей на экран "Шаблоны". Ничего не перезаписывается молча.

### Предупреждение о замене — только когда есть что заменять

Сейчас предупреждение "Это заменит уже запланированные рецепты на этой неделе" показывается всегда при выборе шаблона. Оно должно появляться **только если в текущей открытой неделе уже что-то запланировано**. Если неделя пустая — сразу применяем, без предупреждения. Покажи оба состояния.

Ещё важно по смыслу применения: шаблон перезаписывает только те слоты, которые в нём заполнены. Слоты, которых в шаблоне нет, в неделе остаются нетронутыми — это не полная замена всей недели.

## 3. Импорт рецептов — формат промпта должен совпадать с форматом приложения

Промпт, который генерирует экран импорта, сейчас описывает не тот формат, который приложение умеет принимать (`name`/`meal`/`min`/`ing`, единицы вроде "ломт" и "зуб"). Замени содержимое генерируемого промпта на такой формат:

```json
{
  "recipes": [
    {
      "title": "Куриный суп",
      "baseServings": 4,
      "cookTimeMinutes": 40,
      "cookingMethods": ["stovetop"],
      "ingredients": [
        { "name": "Куриное филе", "quantity": 500, "unit": "g" },
        { "name": "Морковь", "quantity": 1, "unit": "pcs" },
        { "name": "Вода", "quantity": 1500, "unit": "ml" }
      ],
      "steps": [
        "Вскипятить воду, положить курицу целиком",
        "Через 20 минут добавить нарезанные овощи и варить ещё 15 минут"
      ]
    }
  ]
}
```

Что здесь принципиально:
- корневой объект с ключом `recipes`, а не голый массив;
- **нет поля с приёмом пищи** (завтрак/обед/ужин) — рецепт в этом приложении не привязан к слоту;
- `cookingMethods` — только значения `stovetop` / `oven` / `multicooker` / `grill` / `microwave` / `no_cook`, поле необязательное;
- `unit` — только `g` / `ml` / `pcs`, никаких кг, л, ломтиков, зубчиков и головок;
- `steps` — просто строки, без фото.

Плюс в текст промпта добавляется блок со списком названий продуктов из справочника приложения и инструкцией использовать те же названия для совпадающих продуктов. В макете достаточно показать этот блок как несколько строк-примеров с многоточием — важно, что промпт из-за него заметно длиннее, и блок с промптом должен нормально выглядеть при большой высоте (он и так скроллится).

Плейсхолдер в поле "Вставьте JSON сюда" тоже обнови под новый формат.

## 4. Импорт — подтверждение несовпавших ингредиентов как в распознавании фото

Сейчас несовпавший ингредиент — это компактный чип с выпадающим списком "Категория?". Нужно, чтобы он выглядел и работал так же, как строка несовпавшего продукта на экране подтверждения фото холодильника (это тот же по смыслу шаг, и он должен читаться одинаково): строкой на всю ширину, с названием продукта и **двумя вариантами — сопоставить с существующим продуктом из справочника или создать новый с выбором категории**. Сейчас второго варианта ("выбрать существующий") нет вообще.

## 5. Импорт недоступен Участнику

Кнопка "Импортировать" в шапке экрана "Рецепты" сейчас видна всегда. В виде **"Участник"** её быть не должно — импорт создаёт рецепты, а Участник рецепты не создаёт (там же, где скрывается карточка "Добавить рецепт").

Все элементы — в уже установленном стиле прототипа.
