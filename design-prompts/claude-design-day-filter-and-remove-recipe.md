# Промпт-добавка для Claude Design — набор мелких правок по всем экранам

Отправь это следующим сообщением в тот же чат Claude Design — это правки уже существующих экранов ("Список покупок", "Меню на неделю", "Рецепты", "Сегодня", "Инвентарь", "Профиль"), не новые экраны.

---

Доработай следующие экраны в этом прототипе:

**1. Список покупок.** Добавь сверху, над списком, фильтр по дням в виде горизонтальных чипов: "Сегодня", "Завтра", "Вся неделя" (выбран по умолчанию), "Выбрать дни" — последний открывает мультиселект из 7 дней недели (например, шторкой снизу с чекбоксами по дням). Выбор фильтра не меняет визуальную структуру списка (те же категории, те же карточки) — просто сужает, что показано.

**2. Меню на неделю.** У слота дня, в который уже назначен рецепт, добавь заметное действие **"убрать"** — отдельное от "заменить рецепт". Например: маленький крестик/иконка корзины в углу карточки, или свайп влево с кнопкой "Убрать". По нажатию карточка рецепта в слоте исчезает, слот возвращается в пустое состояние с призывом "+ добавить рецепт" (как выглядит ещё не заполненный слот). Сейчас в макете такого действия не было — только выбор рецепта, без возможности его снять.

**3. Рецепты.** Перенеси кнопку/действие "Добавить рецепт" из шапки экрана внутрь самого списка — она должна быть **первой карточкой списка**, перед всеми существующими рецептами (а не иконкой или кнопкой в верхней части экрана). Визуально можно сделать её, например, пунктирной рамкой с "+" и подписью "Добавить рецепт", в том же размере, что и обычные карточки рецептов.

**4. Сегодня — поток "скушал".** Поправь поведение кнопки "скушал": нажатие должно сразу, без какого-либо экрана-подтверждения, помечать приём пищи съеденным (галочка на карточке, лёгкое зачёркивание названия или смена цвета — на твой вкус). Никакого модального окна на этом шаге не нужно. Отдельно, поверх этого действия — покажи необязательную шторку снизу (bottom sheet) "Списать использованные продукты?" со списком ингредиентов рецепта и их количеством (пересчитанным под число порций). Добавь в эту шторку **степпер числа порций** (по умолчанию — сколько было запланировано, но можно поменять, если по факту готовили другое количество) — при изменении список ингредиентов ниже должен пересчитываться. Важно: без сравнения с текущими запасами — не показывай пометки "есть дома"/"нет дома", только сам список "что будет списано" с количеством. Внизу — две одинаковые по весу кнопки: "Списать" и "Пропустить". "Пропустить" просто закрывает шторку без изменений — это нормальный путь, не второсортный. Если этот экран уже сделан иначе (сравнение с запасами, обязательное подтверждение, без степпера порций) — переделай по этому описанию.

**5. Список покупок — категория для ручных позиций.** В форме добавления произвольной (ручной) позиции добавь обязательный выбор категории — простой select/picker (не текстовое поле) со следующими вариантами: Молочные продукты, Мясо и рыба, Овощи и фрукты, Крупы и бакалея, Хлеб и выпечка, Напитки, Заморозка, Бытовая химия, Гигиена и красота, Другое. По умолчанию выбрано "Другое". Это тот же список категорий, что используется для группировки остальных позиций списка — чтобы было явно видно, в какую секцию попадёт добавленная позиция (например, зубная паста).

**6. Сегодня — время приготовления и бейджи.** На карточке каждого приёма пищи добавь примерное время приготовления (например, "~30 мин") и рядом с ним — те же бейджи способа приготовления, что и на карточках экрана "Рецепты" (плита, духовка, мультиварка, гриль, микроволновка, без готовки). Сейчас на этом экране не видно ни времени, ни того, что понадобится для готовки — хочется видеть это сразу, не заходя в сам рецепт.

**7. Рецепты — редактирование и удаление.** У каждой карточки рецепта добавь видимое действие редактирования и удаления — например, свайп влево с двумя кнопками "Изменить"/"Удалить", или меню из трёх точек на карточке. Удаление должно запрашивать подтверждение (это необратимое действие).

**8. Домашние запасы — редактирование и удаление.** У каждой позиции инвентаря добавь возможность отредактировать количество/единицу (например, тап по позиции открывает редактирование) и удалить позицию целиком (например, свайп влево с кнопкой "Удалить").

**9. Список покупок — мгновенная отметка "куплено" + кнопка массового переноса в запасы.** Если сейчас нажатие на чекбокс "куплено" открывает всплывающее окно с предложением добавить в инвентарь — убери это: нажатие должно быть мгновенным (просто галочка/зачёркивание), без всякого окна. Вместо этого добавь отдельную кнопку **"Добавить в запасы"** (сверху или снизу списка) — она видна, когда в списке есть хотя бы одна отмеченная "куплено" позиция. По нажатию открывается экран со списком всех купленных позиций и их количеством, которое можно поправить, а по подтверждению — все они разом добавляются в инвентарь. Также добавь возможность редактировать количество/единицу (и название — для ручных позиций) прямо в списке покупок, не только при первом добавлении.

**10. Профиль — кнопка "Сгенерировать новую ссылку" видна только после "Пригласить".** Сейчас, если обе кнопки ("Пригласить" и "Сгенерировать новую ссылку") показаны сразу — исправь: должна быть видна только кнопка "Пригласить"; по нажатию на неё раскрывается блок с самой ссылкой (и кнопками "Скопировать"/"Поделиться"), и только внутри этого раскрытого состояния появляется вторичная кнопка поменьше "Сгенерировать новую ссылку".

Все изменения — в уже установленном стиле прототипа.
