# Промпт для Claude Design — FridgeMind

Скопируй текст ниже целиком в Claude Design как первое сообщение. Он специально написан так, чтобы Claude Design сам придумал стиль, цвета и композицию — тебе не нужно заранее визуализировать экраны, только функциональные требования уже даны.

---

Спроектируй мобильный интерфейс приложения **FridgeMind** — PWA для планирования питания и учёта продуктов дома. Приложение общее на семью (household) — сколько бы человек в неё ни входило, все делят одни запасы, меню и список покупок, устанавливается на телефон как иконка, используется каждый день дома и в магазине. Дизайни **mobile-first**, под экран телефона (примерно 390×844), не десктоп.

Стиль, цвета, типографику, общую композицию и настроение придумай сам — у меня нет чёткого визуального референса, доверяю тебе. Ориентир по духу: еда, дом, забота о себе, но не приторно — что-то чистое, спокойное, с аппетитными акцентами (например, для фото рецептов/продуктов), без визуального шума, потому что экранами пользуются на бегу (в магазине, на кухне с грязными руками).

Нужно 5 экранов, связанных общим дизайн-языком (одна палитра, одна типографика, одни компоненты кнопок/карточек):

1. **Сегодня** (главный экран/точка входа). Список приёмов пищи на текущий день — это не фиксированный набор, а произвольные слоты, которые пользователь сам создал и назвал (по умолчанию "Завтрак/Обед/Ужин", но могут быть переименованы или дополнены, например, "Перекус"), показанные в заданном пользователем порядке. У каждого приёма пищи — фото и название рецепта, примерное время приготовления (например, "~30 мин") и рядом с ним такие же бейджи способа приготовления, что и на карточках экрана "Рецепты" (плита, духовка, мультиварка и т.д.), и кнопка/действие "скушал". Нажатие сразу отмечает приём пищи как съеденный (без всяких подтверждений, мгновенно — например, галочка или зачёркивание карточки). Отдельно, поверх этого, показывается необязательный экран-предложение "Списать использованные продукты?" — список ингредиентов рецепта с их примерным количеством (пересчитанным под число порций), без какого-либо сравнения с текущими запасами (не показывай тут "есть/нет дома" — это про другой экран). На этом же экране есть степпер числа порций (вдруг готовили не столько, сколько планировали) — при изменении количества ингредиентов ниже пересчитываются. У этого предложения два равнозначных по весу действия: "Списать" (с возможностью поправить цифры) и "Пропустить".

2. **Меню на неделю**. 7 дней, в каждом дне — те же пользовательские слоты приёмов пищи, в том же порядке, что и на экране "Сегодня", в слот выбирается рецепт из библиотеки. При выборе рецепта в слот можно сразу поправить число порций именно на этот раз (степпер, по умолчанию — базовое число порций рецепта). У слота с уже назначенным рецептом должно быть заметное действие "убрать" (не только "заменить") — например, маленький крестик на карточке или свайп — по нажатию слот возвращается в пустое состояние "+ добавить рецепт". Нужен явный переход к экрану "Сегодня"/конкретного дня.

3. **Рецепты**. Список карточек (фото, название), первой карточкой в списке — "+ Добавить рецепт" (не кнопка/иконка в шапке экрана, а именно первый элемент самого списка, перед всеми рецептами). У каждой карточки должно быть видно действие редактирования и удаления (например, свайп или меню из трёх точек) — удаление рецепта должно запрашивать подтверждение. Есть переключатель/фильтр "что приготовить из того, что есть" — рецепты сортируются или помечаются по проценту совпадения ингредиентов с домашними запасами (например, бейдж "80% есть дома"). На каждой карточке также — примерное время приготовления и рядом маленькие бейджи со значками способа приготовления (плита/сковорода, духовка, мультиварка, гриль, микроволновка, без готовки — у рецепта может быть несколько сразу). При просмотре рецепта — степпер числа порций (например, "4 порции" с кнопками −/+), при изменении которого пересчитываются количества всех ингредиентов на экране. Отдельно — экран/форма создания рецепта: название, базовое число порций, выбор бейджей способа приготовления (мультиселект), список ингредиентов с количеством и единицей, и **шаги приготовления по одному** — каждый шаг добавляется отдельной карточкой с текстом и опциональной фотографией именно этого шага (не одно большое текстовое поле). При просмотре готового рецепта шаги пролистываются один за другим, с фото сверху, если оно есть.

4. **Домашние запасы (инвентарь)**. Список продуктов, которые есть дома, сгруппированный по категориям (молочка, овощи, крупы и т.д.), с количеством. Каждую позицию можно отредактировать (изменить количество/единицу) и удалить (например, тап открывает редактирование, свайп — быстрое удаление). Кнопка "добавить вручную" и отдельная кнопка/CTA "сфотографировать холодильник" — по нажатию можно сделать или выбрать не одно фото, а несколько (например, отдельно холодильник, отдельно полка с крупами), с превью-полоской миниатюр внизу экрана камеры и кнопкой "Добавить ещё" / "Готово, распознать"; после отправки показывается экран с одним объединённым распознанным списком продуктов со всех фото сразу, которые нужно подтвердить или поправить перед сохранением.

5. **Список покупок**. Сверху — фильтр по дням: чипы "Сегодня" / "Завтра" / "Вся неделя" (выбрано по умолчанию) / "Выбрать дни" (открывает мультиселект из 7 дней недели). Ниже — список недостающих продуктов на выбранные дни, сгруппированный по категориям (как в магазине: молочка, овощи/фрукты, мясо/рыба, бакалея, напитки, заморозка, бытовая химия, гигиена, другое), с чекбоксами "куплено". Отдельно можно добавить произвольную позицию, не связанную с рецептами (например, зубная паста) — в форме добавления обязательно нужно выбрать категорию из этого же списка (простой select/picker, не текстовое поле), чтобы позиция сразу попала в понятную секцию списка; по умолчанию выбрана категория "Другое". Отметка "куплено" — мгновенная (чекбокс/зачёркивание), без каких-либо всплывающих окон при каждом нажатии — это нужно, чтобы быстро отмечать много позиций подряд прямо в магазине. Каждую позицию также можно отредактировать (название у ручных, количество, единицу) прямо в списке. Отдельной кнопкой сверху или снизу списка — **"Добавить в запасы"**: она видна, когда есть хотя бы одна отмеченная "куплено" позиция, и открывает экран со списком всех купленных позиций с их количеством (можно поправить перед подтверждением), по подтверждению — всё разом переносится в домашние запасы. Список обновляется в реальном времени между всеми членами household — стоит предусмотреть небольшой визуальный сигнал, что список только что изменил кто-то ещё (например, лёгкая подсветка новой/изменённой строки или подпись "добавил(а) [имя]").

6. **Профиль / Настройки** (не в таб-баре — открывается по иконке в шапке любого из 5 экранов). Имя и e-mail текущего пользователя, название семьи (household), список её участников — у каждого имя, e-mail и **бейдж роли** (Организатор / Редактор / Участник). Для пользователя с ролью Организатор рядом с каждым участником (кроме себя) — выпадающий список для смены роли и кнопка "Удалить"; для остальных ролей — просто бейдж без возможности редактирования чужих ролей. Кнопка "Пригласить" — по нажатию раскрывает ссылку с кнопками "Скопировать"/"Поделиться", и только внутри этого раскрытого состояния — вторичная кнопка поменьше "Сгенерировать новую ссылку" (до нажатия "Пригласить" её не видно вообще). Дальше — редактируемый список приёмов пищи с возможностью переименовать, удалить, добавить новый и **перетащить для смены порядка** (drag handle), кнопки "Покинуть семью" и "Выйти из аккаунта".

Продумай также нижнюю навигацию между 5 основными экранами (таб-бар) — это основной способ перемещения по приложению; Профиль — отдельная иконка в шапке, не часть таб-бара.

По итогу собери всё в один связный прототип с общим дизайн-языком, а не 5 разрозненных экранов.

## Как передать это в Claude Code

После того как макет в Claude Design устроит — экспортируй его напрямую в Claude Code (есть встроенный экспорт) и попроси Claude Code сверстать на реальном стеке (Next.js + Tailwind + shadcn/ui) по этому образцу, ориентируясь также на функциональные требования из `CLAUDE.md`.
